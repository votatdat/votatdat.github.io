01 Giới thiệu về Descriptors


Đầu tiên, giả sử rằng chúng ta có vấn đề cần giải quyết như sau: chúng ta muốn class Point2D mà giá trị trục số luôn luôn là số nguyên.
Plain attribute cho x và y không thể đảm bảo điều này, thay vào đó chúng ta dùng property với getter, setter.
Chúng ta có đoạn code như ở dưới: {% highlight python %} class Point2D: @property def x(self): return self._x @x.setter def x(self, value): self._x = int(value) @property def y(self): return self._y @y.setter def y(self, value): self._y = int(value) def __init__(self, x, y): self.x = x self.y = y {% endhighlight %} Đoạn code này khá rườm rà và lặp lại, chúng ta có thể chia thành 2 class như ở dưới: {% highlight python %} class IntegerValue: def get(self): return self._value def set(self, value): self._value = int(value) def __init__(self, value=None): if value: self.set(value) class Point2D: x = IntegerValue() y = IntegerValue() {% endhighlight %} Chúng ta lưu lại với tên OOP08_ex01.py, import và chạy thử, các bạn có thể down file này ở đây. {% highlight python %} >>> from OOP08_ex01 import * >>> p = Point2D() >>> p.x = 100.1 >>> p.x 100.1 {% endhighlight %} Chúng ta thấy là kết quả không đúng, lí do là khi chúng ta gọi p.x = 100.1 thì chúng ta tạo một atrribute x mới có kiểu float, hoàn toàn không cột được x với class IntegerValue. {% highlight python %} >>> type(p) >>> type(p.x) {% endhighlight %} Chúng ta đã thấy ở trên, type(p.x) không trả về 'OOP08_ex01.IntegerValue'.
Chúng ta cần bảo Python làm 2 việc:
  • x = IntegerValue, phải cột biến x vào class này ở run-time.
  • p.x, sử dụng get và set cho instance của class IntegerValue

Đó là lí do Descriptors xuất hiện, chúng ta có 4 method cho descriptor protocol.
  • __get__: để lấy giá trị của attribute, p.x
  • __set__: để set giá trị của attribute, p.x = 100
  • __delete__: để xóa attribute
  • __set_name__: mới xuất hiện ở Python 3.6

Và các descriptor này được chia thành 2 mục chính:
  • __get__: non-data descriptor
  • __set__, __delete__: data descriptor
